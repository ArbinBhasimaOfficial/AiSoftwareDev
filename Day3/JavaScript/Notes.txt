    Variables
        Named references to values.
            const   
                Cannot be reassigned. Prefer by default
                const FullName = "Arbin Bhasima";
            
            let
                Reassignable. Use when the value must change.
                let age = 23;

            var
                var is legacy
                Function-scoped and full of surprises. Only recognise it in old code.

            const vs let
                Reassignement is the only difference beginners need 

                const name = "Arbin Bhasima"
                let currentAge = 23;
                currentAge = 24;
            
                IMPORTANT
                    const does not mean frozen
                    Object properties can still change. Only the binding is locked.


            Naming Variables    
                Names are documentation
                    goode conventions
                        const studentName = "Arbin"
                        const isLoggedIn = true
                        const totalPrice = 35000

                    bad conventions
                        const x = "Arbin"
                        const data = true
                        const temp = 35000

                Convention camelCase - first word lowercase, each next word capitalised

    Primitive Data Types
        Simple, immutable values

            String = "text"
            Number = 42
            Boolean = true / false
            Undefined; 
            Null = Null intentional absence
            BigInt = very large integers 100000000 or more
            Symbol = unique identifier

        Focus for beginners: String - Number - Boolean - Undefined - Null
